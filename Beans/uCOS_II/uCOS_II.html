Important Notes
There are a few things to consider using the uCOS-II bean:
  • The RTOS is using a software interrupt (SWI) to switch tasks. For this the RTOS bean is using an inherited bean SWI. Make sure that the inherited SWI bean is configured to be hooked on the SWI interrupt.
    • For HCS08, HC08, HC12, HCS12 and HCS12X/XGATE use the SWI interrupt vector
    • For ColdFire V1 use the vector number specified in the bean properties. E.g. if the uCOS bean property is set to use Trap Number 14, use vector Vtrap14.
  • The RTOS needs a periodic timer. The RTOS bean is using a shared timer bean. Make sure that the timer period is 20ms, otherwise you need to change your timing.
  • The bean creates all UCOS-II source files. As some files from the UCOS code are including files from the 'system' path using files names in <...>, you need to reflect this in the access paths for the compiler. Otherwise you will get compiler errors that header files like 'ucos_ii.h' cannot be found. Make sure you add into the Access Paths (Project Preference Panel > Target > Access Paths > Systems Paths) following path:
    • {Project}CODE
Project Settings
There are a few points to do manually in order to have the bean working with uCOS-II:
  • For ColdFire V1 make sure you are using following settings for the compiler (Project Preference Panel > Code Generation > ColdFire Processor):
    • Parameter passing: Standard
    • A6 Stack Frames
