Component PDC8544
Display driver for the Philips PDC8544 display driver
Component Level: Low
Category: CPU External Devices-Display
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Properties - Properties of display
    • Width - Width of the display in pixels.
    • Height - The height of the display in pixels.
    • Bytes in rows - If bytes are organized in rows or columns.
    • Bytes in x direction - If bytes are increasing in x direction (from left to right). Otherwise bytes are increasing in y direction (from top to down).
    • MSB first - If the Most Significant bit/pixel comes first in the byte.
    • Bits per pixel - Number of bits per pixel. 1 for B/W displays.
    • Window capability - If the display is able to use a window for writing into display memory.
    • Display Memory Write - If we can write directly into the display memory
  • Type - Defines the specific display type.

    There are 3 options:
    • LPH7366: Nokia 5110, 5120, 5130, 5160, 6110, 6150
    • LPH7677: Nokia 3210, 5510
    • LPH7779: Nokia 3310, 3315, 3330, 3350, 3410

  • Invert Display - If the pixels on the display shall be inverted.
  • Initialize contrast - If disabled, then the driver will still apply a default contrast of 50 for the LPH7366 display type. All other displays know are using a default contrast value of 68.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Contrast - Initial contrast value for the display. Note that LPH7677 & LPH7779 needs typically a value of 68, where LPH7366 needs a value around 50.
  • Mode - If the display is using inverted mode or normal mode.

    There are 2 options:
    • normal: Normal display mode
    • inverted: Inverted display mode

  • Voltage - Specifies the display supply (and logic level) voltage

    There are 2 options:
    • V3: 3.0-3.3V display supply voltage
    • V5: 5.0V display supply voltage

  • HW non-LDD - Group controlling the hardware settings.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • RES - Reset signal pin
    • SCE - SPI Chip Enable/Select signal (low active)
    • D_C - Data/Command mode select pin.
    • SPI - Group to control the SPI.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • SPI - Interface to the SPI communication channel to the display.
  • HW LDD - Group controlling the hardware settings.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • RES - Reset signal pin
    • SCE - SPI Chip Enable/Select signal (low active)
    • D_C - Data/Command mode select pin.
    • SPI - Interface to the SPI communication channel to the display.
  • System - Group controlling the system settings.
    • Wait - Interface for busy waiting.
    • Initialize on Init - If set to 'yes', it will initialize the driver during startup in PE_low_level_init() inside main(). If set to 'no', it is your responsibility to call the component Init() method.
