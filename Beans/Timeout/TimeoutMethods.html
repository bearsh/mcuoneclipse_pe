Component Timeout
Component to manage timeouts, e.g. to be used in loops.
Component Level: High
Category: SW-User Components
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • GetCounter - Initializes a new timeout counter and returns the handle to it. At the end, use LeaveCounter() to free up the resource.

      ANSIC prototype: ComponentName_CounterHandle GetCounter(ComponentName_CounterType nofTicks)
    • nofTicks:ComponentName_CounterType - Number of ticks for the counter until it expires.
    • Return value:ComponentName_CounterHandle - Handle to the counter, to be used for further API calls.

  • LeaveCounter - To be called to return the counter. Note that a counter always should be returned so it can be reused.

      ANSIC prototype: void LeaveCounter(ComponentName_CounterHandle handle)
    • handle:ComponentName_CounterHandle - Counter handle

  • Value - Return the current value of the counter (in ticks)

      ANSIC prototype: ComponentName_CounterType Value(ComponentName_CounterHandle handle)
    • handle:ComponentName_CounterHandle - Handle of the timeout counter
    • Return value:ComponentName_CounterType - Returns the value of the timeout counter.

  • SetCounter - Sets the counter to a new value and returns the value just prior to the call.

      ANSIC prototype: ComponentName_CounterType SetCounter(ComponentName_CounterHandle handle, ComponentName_CounterType nofTicks)
    • handle:ComponentName_CounterHandle - Counter handle which shall get a new value.
    • nofTicks:ComponentName_CounterType - New value (tick count) of the timeout counter. Pass zero to have it expire immediately.
    • Return value:ComponentName_CounterType - Value of counter before reset.

  • CounterExpired - Returns true if the timeout counter has been expired

      ANSIC prototype: bool CounterExpired(ComponentName_CounterHandle handle)
    • handle:ComponentName_CounterHandle - The timeout handle retrieved using GetCounter()
    • Return value:bool - Returns TRUE if the counter has been expired, FALSE otherwise

  • AddTick - Method to be called from a periodic timer or interrupt. It will decrement all current counters by one down to zero.

      ANSIC prototype: void AddTick(void)

  • Init - Initialization of the driver

      ANSIC prototype: void Init(void)


Note: Some methods can be implemented as macros.
