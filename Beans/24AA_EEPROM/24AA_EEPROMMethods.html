Component 24AA_EEPROM
Microchip 24AA Serial EEPROM
Component Level: Low
Category: CPU External Devices-Memory
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • ReadByte - Reads a single byte from the given memory address

      ANSIC prototype: byte ReadByte(ComponentName_Address addr, byte *data)
    • addr:ComponentName_Address - The address where to read from memory.
    • data: Pointer to byte - Pointer to a location where to store the data
    • Return value:byte - Error code, possible values
      ERR_OK - OK
      otherwise it can return an error code of the underlying communication protocol.

  • WriteByte - Writes a single byte to specified address

      ANSIC prototype: byte WriteByte(ComponentName_Address addr, byte data)
    • addr:ComponentName_Address - The address inside the EEPROM
    • data:byte - The data value to write
    • Return value:byte - Error code, possible values
      ERR_OK - OK
      otherwise it can return an error code of the underlying communication protocol.

  • ReadBlock - Read a block of memory.

      ANSIC prototype: byte ReadBlock(ComponentName_Address addr, byte *data, word dataSize)
    • addr:ComponentName_Address - Address where to read the memory
    • data: Pointer to byte - Pointer to a buffer where to store the data
    • dataSize:word - Size of buffer the data pointer is pointing to
    • Return value:byte - Error code, possible values
      ERR_OK - OK
      otherwise it can return an error code of the underlying communication protocol.

  • WriteBlock - Writes a block of data to the EEPROM

      ANSIC prototype: byte WriteBlock(ComponentName_Address addr, byte *data, word dataSize)
    • addr:ComponentName_Address - Address of memory
    • data: Pointer to byte - Pointer to the data
    • dataSize:word - Size of data
    • Return value:byte - Error code, possible values
      ERR_OK - OK
      ERR_OVERFLOW - data block passed has either size of zero or exceeds internal buffer size
      otherwise it can return an error code of the underlying communication protocol.

  • WriteProtect - Prevents writing to the memory. Read operations are not affected.

      ANSIC prototype: void WriteProtect(void)

  • WriteUnprotect - Allows writing to the memory

      ANSIC prototype: void WriteUnprotect(void)

  • SelectDevice - Configures the I2C address to be used for the memory. With this multiple devices attached to the bus can be addressed.

      ANSIC prototype: byte SelectDevice(byte addrI2C)
    • addrI2C:byte - I2C Address of the memory device, formed by the address pins.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_VALUE - address exceeds address pins of device type used

  • GetSelectedDevice - Returns the currently used I2C address (e.g. set with SelectDevice()).

      ANSIC prototype: byte GetSelectedDevice(void)
    • Return value:byte - I2C device address

  • Test - Test routine to test the driver. Note that this routine writes to the EEPROM!

      ANSIC prototype: byte Test(void)
    • Return value:byte - Error code, possible values
      ERR_OK - OK
      ERR_FAULT - Test failed

  • ParseCommand - Shell Command Line parser. This method is enabled/disabled depending on if you have the Shell enabled/disabled in the properties.

      ANSIC prototype: byte ParseCommand(const unsigned char *cmd, bool *handled, const %@Shell@'ModuleName'%.StdIOType *io)
    • cmd: Pointer to byte - Pointer to command string
    • handled: Pointer to bool - Pointer to variable which tells if the command has been handled or not
    • io: Pointer to byte - Pointer to I/O structure
    • Return value:byte - Error code


Note: Some methods can be implemented as macros.
