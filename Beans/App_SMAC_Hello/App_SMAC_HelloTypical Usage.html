Component App_SMAC_Hello
Simple SMAC 'Hello' Application
Component Level: High
Category: SW-User Components
Typical Usage:
(Examples of a typical usage of the component in user code. For more information please see the page Component Code Typical Usage.)

Typically there is one sender and one receiver. The sender is sending a 'hello' message, and the receiver acknowledges the message with an 'ack' message. At startup each device toggles for a short time all 4 LEDs. If communication is happening correctly, then each device toogles one LED (LED1). If the sender does not receive a response, then it will toggle two LEDs (LED2 and LED3) instead.

Make sure after you added the bean to your project that all settings are correct and corresponding to your application. If you do not have an LCD display, you can disable it in the bean settings. Make sure that you have especially the correct settings for the SMAC portion (see help for the SMAC bean).

Possible settings for the Freescale SRB (802.15.4 Sensor Reference board):
- All LEDs: Anode on port side: no
- LED1: D4
- LED2: D5
- LED3: D6
- LED4: D7
- LCD: disabled
