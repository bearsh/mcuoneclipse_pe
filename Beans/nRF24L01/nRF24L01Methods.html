Component nRF24L01
Driver for the Nordic nRF241L01+ Wireless Transceiver
Component Level: Low
Category: CPU External Devices-Communication
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • WriteRegister - Write a register value to the transceiver

      ANSIC prototype: void WriteRegister(byte reg, byte val)
    • reg:byte - Register address to write
    • val:byte - Value to write

  • ReadRegister - Reads a register from the transceiver

      ANSIC prototype: byte ReadRegister(byte reg)
    • reg:byte - Register address
    • Return value:byte - value read

  • ReadRegisterData - Read multiple bytes from the bus.

      ANSIC prototype: void ReadRegisterData(byte reg, byte *buf, byte bufSize)
    • reg:byte - Register address
    • buf: Pointer to byte - Pointer to buffer where to store the values
    • bufSize:byte - Size of the destination buffer

  • WriteRegisterData - Write multiple bytes to the bus.

      ANSIC prototype: void WriteRegisterData(byte reg, byte *buf, byte bufSize)
    • reg:byte - Register address
    • buf: Pointer to byte - Pointer to data buffer to write
    • bufSize:byte - Size of buffer in bytes

  • WriteRead - Writes a byte and reads back one byte

      ANSIC prototype: byte WriteRead(byte val)
    • val:byte - Value to write to the bus
    • Return value:byte - Value read from the bus

  • Write - Writes a byte to the bus, without returning the byte read.

      ANSIC prototype: void Write(byte val)
    • val:byte - Value to write

  • GetStatus - Returns the device status byte

      ANSIC prototype: byte GetStatus(void)
    • Return value:byte - status byte

  • GetStatusClrIRQ - Returns the device status byte and clears any IRQ bits in the status register.

      ANSIC prototype: byte GetStatusClrIRQ(void)
    • Return value:byte - status byte

  • ResetStatusIRQ - Reset the given flags mask of status bits

      ANSIC prototype: void ResetStatusIRQ(byte flags)
    • flags:byte - Flags, one or more of RF24_STATUS_RX_DR, RF24_STATUS_TX_DS, RF24_STATUS_MAX_RT

  • TxPayload - Send the payload to the Tx FIFO and send it

      ANSIC prototype: void TxPayload(byte *payload, byte payloadSize)
    • payload: Pointer to byte - Pointer to buffer with payload to send
    • payloadSize:byte - Size of payload buffer

  • RxPayload - Receive the Rx payload from the FIFO and stores it in a buffer.

      ANSIC prototype: void RxPayload(byte *payload, byte payloadSize)
    • payload: Pointer to byte - Pointer to the payload buffer
    • payloadSize:byte - Size of the payload buffer

  • StopRxTx - Stops sending or receiving (sets CE pin to LOW).

      ANSIC prototype: void StopRxTx(void)

  • StartRxTx - Starts sending or receiving (sets CE pin to HIGH).

      ANSIC prototype: void StartRxTx(void)

  • EnableAutoAck - Enables Auto-Acknowledgment for the given pipe(s) (RF_EN_AA register).

      ANSIC prototype: byte EnableAutoAck(byte pipes)
    • pipes:byte - mask of pipes to be enabled (ENAA_P0 to ENAA_P5)
    • Return value:byte - Error code

  • SetStaticPipePayload - Specifies the static payload for a pipe.

      ANSIC prototype: byte SetStaticPipePayload(byte pipe, byte payloadBytes)
    • pipe:byte - Pipe number, 0 to 5
    • payloadBytes:byte - Number of payload pipes
    • Return value:byte - Error code

  • EnableDynanicPayloadLength - Enables dynamic payload length for the give pipes

      ANSIC prototype: byte EnableDynanicPayloadLength(byte pipeMask)
    • pipeMask:byte - Mask of pipes, with 1 for pipe 0, 2 for pipe 1, 4 for pipe 2 and so on.
    • Return value:byte - Error code

  • WriteFeature - Writes the FEATURE register

      ANSIC prototype: byte WriteFeature(byte featureMask)
    • featureMask:byte - Mask of FEATURE, e.g. FEATURE_EN_DPL
    • Return value:byte - Error code

  • ReadFeature - Reads the FEATURE register

      ANSIC prototype: byte ReadFeature(byte *featureMask)
    • featureMask: Pointer to byte - Pointer to value of FEATURE register
    • Return value:byte - Error code

  • ReadNofRxPayload - Returns the number of received payload bytes for the top R_RX_PAYLOAD in the RX FIFO

      ANSIC prototype: byte ReadNofRxPayload(byte *nof)
    • nof: Pointer to byte - Pointer to where to store the number of bytes. A return value of 0 means that the pipe is not used, and if the value is >32 then the data is not valid.
    • Return value:byte - Error code

  • ReadObserveTxRegister - Reads the OBSERVE_TX register to return the count of loss packets and count of retransmitted packets.

      ANSIC prototype: byte ReadObserveTxRegister(byte *nofLoss, byte *nofRetransmitted)
    • nofLoss: Pointer to byte - Pointer to number of lost packets
    • nofRetransmitted: Pointer to byte - Pointer to number of retransmitted packets
    • Return value:byte - Error code

  • ReadReceivedPowerDetector - Reads the RPD (Received Power Detector) register.

      ANSIC prototype: byte ReadReceivedPowerDetector(byte *rpd)
    • rpd: Pointer to byte - Pointer to RPD bit. Bit is one for received power levels above -64 dBm, zero for less than -64 dBm.
    • Return value:byte - Error code

  • SetChannel - Sets the radio channel (RF_CH register).

      ANSIC prototype: byte SetChannel(byte channel)
    • channel:byte - Channel number, valid channel numbers are 0x0 to 0x7F
    • Return value:byte - Error code

  • GetChannel - Returns the radio channel (RF_CH register).

      ANSIC prototype: byte GetChannel(byte *channel)
    • channel: Pointer to byte - Pointer to where to store the channel number.
    • Return value:byte - Error code

  • ConstantCarrierWave - Put the transceiver into contant carrier wave output for testing. The output power of a radio is a critical factor for achieving wanted range. Output power is also the first test

      ANSIC prototype: byte ConstantCarrierWave(bool turnOn)
    • turnOn:bool - Set to true to start constant carrier wave, false to stop it.
    • Return value:byte - Error code

  • SetOutputPower - Sets the output power

      ANSIC prototype: byte SetOutputPower(signed char power)
    • power:signed char - Ouput power in dBm, either 0, -10, -12 or -18
    • Return value:byte - Error code

  • GetOutputPower - Returns the current output power

      ANSIC prototype: byte GetOutputPower(signed char *power)
    • power: Pointer to signed char - Pointer to where to store the value. Returns the output power in dBm
    • Return value:byte - Error code

  • Init - Initialization method

      ANSIC prototype: void Init(void)


Note: Some methods can be implemented as macros.
