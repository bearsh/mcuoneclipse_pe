Component MMA7260Q
Driver for the Freescale 3-Axis Accelerometer MMA7260Q
Component Level: Low
Category: CPU External Devices-Sensor
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • GetX - Retrieves the value for the X axis. The value is adjusted with the zero calibration value (0 for 0 g, negative for negative acceleration and positive for positive acceleration).

      ANSIC prototype: int16_t GetX(void)
    • Return value:int16_t - Measured X value

  • GetY - Retrieves the value for the Y axis. The value is adjusted with the zero calibration value (0 for 0 g, negative for negative acceleration and positive for positive acceleration).

      ANSIC prototype: int16_t GetY(void)
    • Return value:int16_t - Measured Y value

  • GetZ - Retrieves the value for the Z axis. The value is adjusted with the zero calibration value (0 for 0 g, negative for negative acceleration and positive for positive acceleration).

      ANSIC prototype: int16_t GetZ(void)
    • Return value:int16_t - Measured Z value

  • Gselect - Allows to select different g sensitivity levels.

      ANSIC prototype: void Gselect(byte gSelect)
    • gSelect:byte - Pass 0 for 1.5g, 1 for 2g, 3 for 4g and 4 for 6g sensitivity. The driver header file defines the values as macros as well.

  • SensorOn - Enables the sensor.

      ANSIC prototype: void SensorOn(void)

  • SensorOff - Disables the sensor (puts it in sleep mode).

      ANSIC prototype: void SensorOff(void)

  • CalibrateX1g - Performs a calibration of the sensor. It is assumed that the Y and Z sensors have 0 g, and the X sensor has 1 g.

      ANSIC prototype: void CalibrateX1g(void)

  • CalibrateY1g - Performs a calibration of the sensor. It is assumed that the X and Z sensors have 0 g, and the Y sensor has 1 g.

      ANSIC prototype: void CalibrateY1g(void)

  • CalibrateZ1g - Performs a calibration of the sensor. It is assumed that the X and Y sensors have 0 g, and the Z sensor has 1 g.

      ANSIC prototype: void CalibrateZ1g(void)

  • GetXmg - Returns the X value in mg

      ANSIC prototype: int16_t GetXmg(void)
    • Return value:int16_t - The g value in 1/1000g units

  • GetYmg - Returns the Y value in mg

      ANSIC prototype: int16_t GetYmg(void)
    • Return value:int16_t - The g value in 1/1000g units

  • GetZmg - Returns the Z value in mg

      ANSIC prototype: int16_t GetZmg(void)
    • Return value:int16_t - The g value in 1/1000g units

  • MeasureGetRawX - Performs a measurement on X channel and returns the raw value.

      ANSIC prototype: word MeasureGetRawX(void)
    • Return value:word - X sensor value

  • MeasureGetRawY - Performs a measurement on Y channel and returns the raw value.

      ANSIC prototype: word MeasureGetRawY(void)
    • Return value:word - Y sensor value

  • MeasureGetRawZ - Performs a measurement on Z channel and returns the raw value.

      ANSIC prototype: word MeasureGetRawZ(void)
    • Return value:word - Z sensor value

  • GetXOffset - Returns the offset applied to the X value.

      ANSIC prototype: int16_t GetXOffset(void)
    • Return value:int16_t - Z offset value

  • GetYOffset - Returns the offset applied to the Y value.

      ANSIC prototype: int16_t GetYOffset(void)
    • Return value:int16_t - Z offset value

  • GetZOffset - Returns the offset applied to the Z value.

      ANSIC prototype: int16_t GetZOffset(void)
    • Return value:int16_t - Z offset value

  • GetX1gValue - Returns the value for 1g for channel X.

      ANSIC prototype: int16_t GetX1gValue(void)
    • Return value:int16_t - 1g value for X

  • GetY1gValue - Returns the value for 1g for channel Y.

      ANSIC prototype: int16_t GetY1gValue(void)
    • Return value:int16_t - 1g value for Y

  • GetZ1gValue - Returns the value for 1g for channel X.

      ANSIC prototype: int16_t GetZ1gValue(void)
    • Return value:int16_t - 1g value for Y

  • Deinit - Deinitializes the driver

      ANSIC prototype: byte Deinit(void)
    • Return value:byte - Error code

  • ParseCommand - Shell Command Line parser. This method is enabled/disabled depending on if you have the Shell enabled/disabled in the properties.

      ANSIC prototype: byte ParseCommand(const unsigned char *cmd, bool *handled, const %@Shell@'ModuleName'%.StdIOType *io)
    • cmd: Pointer to byte - Pointer to command string
    • handled: Pointer to bool - Pointer to variable which tells if the command has been handled or not
    • io: Pointer to byte - Pointer to I/O structure
    • Return value:byte - Error code


Note: Some methods can be implemented as macros.
