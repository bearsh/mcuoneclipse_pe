Component LCDHTA
Character LCD Display
Component Level: High
Category: CPU External Devices-Display
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • Write - Write a single character to the display

      ANSIC prototype: void Write(char ch)
    • ch:char - Character to write

  • WriteLn - Writes a new line to the display

      ANSIC prototype: void WriteLn(void)

  • WriteLineStr - Writes a full line to the display (clears the rest of the line).

      ANSIC prototype: void WriteLineStr(byte line, char *str)
    • line:byte - Line number (starting with 1).
    • str: Pointer to char - Pointer to the string which should be shown on the display.

  • WriteString - Writes a string to the display at the current cursor position.

      ANSIC prototype: void WriteString(char *str)
    • str: Pointer to char - Pointer to string (zero byte terminated) to write to the display

  • LoadSoftChar - Loads a user defined (softchar) into the display CGRAM.

      ANSIC prototype: void LoadSoftChar(byte charCode, byte *softChar)
    • charCode:byte - The character code to be defined (0..7)
    • softChar: Pointer to byte - Pointer to an array of 8 bytes defining the soft character
      Example of the soft character 'ü':
      const byte SoftCharUE[8] = { // ü
      0x11, 0x00, 0x11, 0x11, 0x11, 0x13, 0x0d, 0
      // X...X
      // .....
      // X...X
      // X...X
      // X...X
      // X..XX
      // .XX.X
      };

  • ShiftLeft - Shifts all characters to the left.

      ANSIC prototype: void ShiftLeft(void)

  • ShiftRight - Shifts all characters to the right.

      ANSIC prototype: void ShiftRight(void)

  • GotoXY - Places the cursor on a specified position on the display.

      ANSIC prototype: void GotoXY(byte line, byte column)
    • line:byte - Line number starting with 1
    • column:byte - Column number starting with 1

  • SetEntryMode - Configures the display entry mode, if the cursor has to shift and/or if the display shall shift content while displaying text.

      ANSIC prototype: void SetEntryMode(bool increment, bool shiftLeft)
    • increment:bool - Increments (TRUE) or decrements (FALSE) the display address by 1 when a character code is written into or read from DDRAM. The cursor or blinking moves to the right when incremented by 1 and to the left when decremented by 1.
    • shiftLeft:bool - The display does not shift if the 'shift' is FALSE. If 'shift' is TRUE, it will seem as if the cursor does not move but the display does.

  • DisplayOn - Sends the display on command to the display.

      ANSIC prototype: void DisplayOn(void)

  • DisplayOff - Sends the display off command to the display.

      ANSIC prototype: void DisplayOff(void)

  • CursorOn - Enables the cursor.

      ANSIC prototype: void CursorOn(void)

  • CursorOff - Disables the cursor.

      ANSIC prototype: void CursorOff(void)

  • BlinkingOn - Sends the display off command to the display.

      ANSIC prototype: void BlinkingOn(void)

  • BlinkingOff - Puts the display in blinking off mode.

      ANSIC prototype: void BlinkingOff(void)

  • Home - Moves the cursor to the beginning of the first line.

      ANSIC prototype: void Home(void)

  • Line - Sets the current line.

      ANSIC prototype: void Line(byte line)
    • line:byte - Line number, starting with 1

  • Clear - Clears the display and moves the cursor to the first line.

      ANSIC prototype: void Clear(void)

  • UseDisplay - no hint

      ANSIC prototype: byte UseDisplay(byte display)
    • display:byte - Has to be either 1 (top display, using E1) or 2 (bottom display, using E2)
    • Return value:byte - Error code


Note: Some methods can be implemented as macros.
