Component LCDHTA
Character LCD Display
Component Level: High
Category: CPU External Devices-Display
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • HW Interface - Defines how the display is connected to the controller.
    • LCD Type - Defines the specific LCD type, as some devices may have special timing needs.

      There are 2 options:
      • generic: Generic protocol, shall work for most displays.
      • Displaytech 162c: For the Displaytech 162c display, there is a longer EN pulse required (slower timing).

    • LCD Lines - Number or lines on display.
    • Characters per Line - Number of characters per line
    • LCD Enable Signal - If the display has a select signal to choose a display on the bus.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • LCD Enable signal - Display RS pin.
    • Read from Display - If you can (and want) read from the display data bus. This controls the R/W line (if present).
      The following items are available only if the group is enabled (the value is "Enabled"):
      • R/W signal - Display R/W pin. This pin controls if data is written to or read from the LCD. The signal is high for reading, low for only writing to the display.
      • Check Busy Flag - If the software shall check the LCD busy state.Some display may not provide this information. Note that this requires reading from the display data lines (which may or may not possible with your hardware setup).
    • Wait (us) - Wait for a specific time before writing to the display. Useful if the display has no R/W line and communication to the display is too fast. This emulates the 'Check for busy flag', but not using the R/W pin. A value of 0 disables it.
    • E signal - Display Enable pin. With the E pin data/command read/write cycles are started.
    • E2 - Useful for special 4 line displays which have two display controllers with different enable signals. With this it is possible to control upper and lower half of display. Use the method UseDisplay() to switch the display controller.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • E2 signal - E2 signal of displays which use this signal to control the upper and lower half of display.
    • RS signal - Display RS pin. With the RS (Register Select) pin the display switches between commands (RS=0) and data mode (RS=1).
    • Data/Control Bus - Defines the data & control bus between the display and the controller.
      • Data/Control Bus Width - Width in bits of the data and control bus
      • DB0..DB3 - DB0 to DB3 pins. If you enable DB0..DB3, you need to enable DB4..DB7 as well (8bit bus)
        The following items are available only if the group is enabled (the value is "Enabled"):
        • DB0 - DB0 pin
        • DB1 - DB1 pin
        • DB2 - DB2 pin
        • DB3 - DB3 pin
      • DB4..DB7 - DB4 to DB7 pins. Only enabling the DB4 to DB7 means a 4bit data/control bus
        The following items are available only if the group is enabled (the value is "Enabled"):
        • DB4 - DB4 pin
        • DB5 - DB5 pin
        • DB6 - DB5 pin
        • DB7 - DB7 pin
      • Bits/Byte Bus - non-LDD (Logical Device Driver) Interfaces
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Data/Control Bus - Display data port (4 or 8bit). In 4bit mode you need to connect DB7-DB4
  • System Interface - Defines the interface to the surrounding system.
    • Wait - Interface to the wait (busy wait) interface.
