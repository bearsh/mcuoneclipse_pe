Component FAT_FileSystem
FAT file system module
Component Level: High
Category: SW-Memory Manager
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • FatFs Version - Indentifies the version of FatFs used
  • Tiny - If the file system shall be really small and tiny. This corresponds to the _FS_TINY flag. If enabled, FatFs uses the sector buffer in the file system object instead of the sector buffer in the individual file object for file data transfer. This reduces memory consumption 512 bytes each file object.
  • Volumes - Number of volumes (logical drives) to be used.
  • FS_MINIMIZE - The _FS_MINIMIZE option defines minimization level to remove some functions.
    0: Full function.
    1: f_stat, f_getfree, f_unlink, f_mkdir, f_chmod, f_truncate and f_rename are removed.
    2: f_opendir and f_readdir are removed in addition to 1.
    3: f_lseek is removed in addition to 2.
  • Maximum Sector Size - Specifies the maximum sector size (in bytes) to be handled. This number affects the buffer size allocated in FatFS, so the higher the number, the more memory is used. This controls the _MAX_SS macro. When _MAX_SS > 512, GET_SECTOR_SIZE must be implemented to disk_ioctl().

    There are 4 options:
    • 512: Sector size of 512 bytes, usually for memory card and floppy disk
    • 1024: Sector size of 1024 bytes, usually for floppy disk
    • 2048: Sector size of 2048 bytes, usually for MO disk
    • 4096: Sector size of 4096 bytes.

  • Relative Path - This option configures the relative path feature. It sets the _FS_RPATH variable.

    There are 3 options:
    • Disabled: Disable relative path feature and remove related functions.
    • Enabled: Enable relative path. f_chdrive() and f_chdir() are available.
    • Enabled with f_getcwd(): f_getcwd() is available in addition to 'Enabled'

  • Code Page - The _CODE_PAGE specifies the OEM code page to be used on the target system. Incorrect setting of the code page can cause a file open failure.

    There are 26 options:
    • ASCII (for only non-LFN cfg): ASCII (for only non-LFN cfg) (1)
    • U.S. (OEM): U.S. (OEM) (437)
    • Arabic: Arabic (OEM) (720)
    • Greek (OEM): Greek (OEM) (737)
    • Baltic (OEM): Baltic (OEM) (775)
    • Multilingual Latin 1 (OEM): Multilingual Latin 1 (OEM) (850)
    • Turkish (OEM): Turkish (OEM) (857)
    • Latin 2 (OEM): Latin 2 (OEM) (852)
    • Cyrillic (OEM): Cyrillic (OEM) (855)
    • Multilingual Latin 1 + Euro (OEM): Multilingual Latin 1 + Euro (OEM) (858)
    • Hebrew (OEM): Hebrew (OEM) (862)
    • Russian (OEM): Russian (OEM)
    • Thai (OEM, Windows): Thai (OEM, Windows) (874)
    • Japanese Shift-JIS: Japanese Shift-JIS (932)
    • Simplified Chinese GBK: Simplified Chinese GBK (936)
    • Korean: Korean (949)
    • Traditional Chinese Big5: Traditional Chinese Big5 (950)
    • Central Europe (Windows): Central Europe (Windows) (1250)
    • Cyrillic (Windows): Cyrillic (Windows) (1251)
    • Latin 1 (Windows): Latin 1 (Windows) (1252)
    • Greek (Windows): Greek (Windows) (1253)
    • Turkish (Windows): Turkish (Windows) (1254)
    • Hebrew (Windows): Hebrew (Windows) (1255)
    • Arabic (Windows): Arabic (Windows) (1256)
    • Baltic (Windows): Baltic (Windows) (1257)
    • Vietnam (OEM, Windows): Vietnam (OEM, Windows) (1258)

  • File Sharing - Enables the file sharing feature if value is greater than zero. The value defines how many files can be open simultaneously. This option sets _FS_SHARE.
  • Multipartion - When _MULTI_PARTITION is disabled, each volume is bound to the same physical drive number and can mount only first primaly partition. When it is enabled, each volume is tied to the partitions listed in Drives[].
  • Fast Seek - Enable or disable the fast seek feature. This option sets the _USE_FASTSEEK define.
  • Use Erase - To enable sector erase feature, set _USE_ERASE to 1.
  • String Functions - Configures string functions (macro _USE_STRFUNC in ffconf.h)

    There are 3 options:
    • disable: disables f_putc(), f_puts(), f_printf() and f_get(s).
    • enable: enables f_putc(), f_puts(), f_printf() and f_get(s).
    • enable with CR->LF conversion: same as enable, but with LR->CRLF conversion

  • LFN - Long File Name (LFN)
    • Use LFN - Configures how LFN (Long File Names) shall be used

      There are 4 options:
      • Disable: Disable LFN feature. _MAX_LFN and _LFN_UNICODE have no effect.
      • Enable, static global buffer: Enable LFN with static working buffer on the BSS. Always NOT reentrant.
      • Enable, dynamic stack buffer: Enable LFN with dynamic working buffer on the STACK.
      • Enable, dynamic heap buffer: Enable LFN with dynamic working buffer on the HEAP.

    • Max LFN Length - Maximum LFN length to handle
    • LFN Unicode - If for long file names the names shall be ANSI/OEM (no) or in unicode (yes)
  • Write enabled - If disabled, the driver is in read only mode. This corresponds to the _FS_READONLY flag.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Use TimeDate - If enabled, it uses the TimeDate component to provde time and date information to the FatFS. If disabled, you need to provide your own implementation.
      You need to implement your own 'DWORD get_fattime (void)', see the implementation example in diskio.c
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Realtime clock - Interface to real time clock. Needed for time file time
  • Reentrant - If the driver shall be reentrant. This defines the _FS_REENTRANT define.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Timeout ticks - Timeout period in unit of time ticks of the OS
    • Handle - Type of sync object used on the OS. e.g. HANDLE, OS_EVENT*, ID and etc. If you are enabling an RTOS, then this handle is replaced by the one of the RTOS
    • User Sync Functions - If set to 'yes', you have to provide your own ff_cre_syncobj(), ff_del_syncobj(), ff_req_grant() and ff_rel_grant() functions. Otherwise the component will provide an implementation.
    • RTOS - Configures the RTOS to be used. If enabled, it will use an RTOS synchronization method (e.g. semaphore) to protect access to the file system object.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • RTOS - RTOS to be used
      • Use dynamic heap - If dynamic memory (malloc(), free()) of the RTOS shall be used. This will reduce the amount of stack and global memory used, but requires memory from the operating system.
  • Memory - Interface for the underlying memory (e.g. SD Card)
  • Shell - If enabled, the component offers a command line shell interface
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Shell - Interface to command line shell
    • Utility - Interface to utility functions
    • File print buffer size - Size of read buffer on stack for PrintFile() method: a higher number increases performance, but increases memory used on the stack.
    • File copy buffer size - Size of copy buffer on stack for CopyFile() method: a higher number increases performance, but increases memory used on the stack.
