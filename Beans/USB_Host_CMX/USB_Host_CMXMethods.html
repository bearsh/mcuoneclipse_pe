Component USB_Host_CMX
USB Host driver (based on CMX)
Component Level: Low
Category: SW-Communication
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • enable_usb_pull_downs - Enables USB bus pull-downs

      ANSIC prototype: void enable_usb_pull_downs(void)

  • Vbus_Sw_En - Initialize Vbus switch control, enable VBUS 5V

      ANSIC prototype: void Vbus_Sw_En(void)

  • init_clock - Initializes the bus to the 24 MHz required by USB. Disable this function if your application already cares about this.

      ANSIC prototype: void init_clock(void)

  • hw_init - no hint

      ANSIC prototype: void hw_init(void)

  • host_init - Initializes the USB host mode

      ANSIC prototype: void host_init(void)

  • ByteRev32 - Performs a byte reverse on a 32bit value

      ANSIC prototype: dword ByteRev32(dword val)
    • val:dword - no hint
    • Return value:dword - reverted value

  • stack_init - Stack initialization routine, to be used with stack_size().

      ANSIC prototype: void stack_init(byte pattern)
    • pattern:byte - no hint

  • stack_size - Determines the available stack size. You need to call stack_init() first.

      ANSIC prototype: long stack_size(byte pattern)
    • pattern:byte - no hint
    • Return value:long - stack size

  • ScanDevices - Scans the USB port for devices. If a supported device is detected, it will generate the device events. On detach the method will return.

      ANSIC prototype: byte ScanDevices(void)
    • Return value:byte - Error code

  • Init - Main initalization routine. Call this before any other function.

      ANSIC prototype: byte Init(void)
    • Return value:byte - Error code


Note: Some methods can be implemented as macros.
