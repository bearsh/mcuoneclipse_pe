Component SD_Card
SD Card
Component Level: Low
Category: CPU External Devices-Memory
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Block size - Block size to be used to read/write from/to the device. Only a power of two is allowed. This defines the block size the driver can read in one chunk. If the SD card is used in raw/block mode, then this defines the block size on the device too.

    There are 4 options:
    • 512: block size of 512 bytes (default, for up to 2 GB cards)
    • 1024: block size of 1024 bytes, for cards > 2 GB
    • 2048: block size of 2048 bytes
    • 4096: block size of 4096 bytes

  • Cmd wait counter - When sending a command to the SD card, the driver is waiting for a specified time for the proper response. To poll the response, a counter is used which can be customized with this setting. Lowering this number improves performance, but may result in inproper operation.
  • Wait Ready Timeout (ms) - Wait timeout in milliseconds to wait until the card is ready.
  • Wait Cmd Timeout (ms) - Wait timeout in milliseconds for commands sent to the SD card
  • Receive Block Timeout (ms) - Wait timeout in milliseconds to wait while reading a data block
  • SPI Block Transfer - If set to 'yes' the SPI driver will use block transfer for better performance. This is enabled by SPI_WRITE_BLOCK_ENABLED in the driver.
  • Hardware - Hardware options
    • SW SPI - If enabled, it is using a software SPI connection. Note that a software SPI implementation is slower compared to a hardware SPI
      The following items are available only if the group is enabled (the value is "Enabled"):
      • SPI - SPI communication interface
    • HW SPI - If enabled, it is using a hardware SPI connection. A hardware SPI implementation is faster compared to a sofware SPI implemenation
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Slow Baud Rate Mode - Defines which SPI baud rate mode shall be used during Init. 0 denotes the initial baud rate specified in the SPI configuration component. (max 375 kHz)
      • Fast Baud Rate Mode - Defines which SPI baud rate mode shall be used in InitCommChannelMethod(). 0 denotes the initial baud rate specified in the SPI configuration component. (max 12 MHz)
      • LDD HW SPI - If enabled, a LDD (Logical Device Driver) HW SPI interface is used
        The following items are available only if the group is enabled (the value is "Enabled"):
        • LDD HW SPI - Interface to the Logical Device Driver SPI
      • non-LDD HW SPI - If enabled, a non-LDD (traditional) HW SPI interface is used
        The following items are available only if the group is enabled (the value is "Enabled"):
        • HW SPI - SPI communication interface
    • SPI Read/Write Macros - If enabled, mcros for optimization purpuse to read and write/read from the SPI bus can be provided. With this access to the SPI bus can be optimized.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • SPI Macros - Macros
    • Activate SPI bus - If multiple devices are using the same SPI bus, and if the SPI bus protocol is not the same, then this option needs to by set to 'yes'. This ensures that the bus protocol is changed properly.
    • Slave Select - Pin to select slave
      The following items are available only if the group is enabled (the value is "Enabled"):
      • LDD SS - Logical Device Driver Slave/Chip Select Signal
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Slave Select Pin - Logical Device Driver for Slave/Chip select pin
      • non-LDD SS - Non-Logical Device Driver Slave/Chip Select Signal
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Slave Select pin - Device (Slave) select pin
    • Activate - Pin used to activate memory card bus driver. This is required if you are using the device on the same bus with other devices.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • LDD Activate - Logical Device Driver Activate interface
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Activate pin - If using the memory card with a tristate bus driver, then this needs to be activated. A high level on this pin activates the bus for the memory card.
      • non-LDD Activate - classic (non Logical Device Driver) Activate interface
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Activate pin - If using the memory card with a tristate bus driver, then this needs to be activated. A high level on this pin activates the bus for the memory card.
    • Card detection - A pin can be used to detect if a card is inserted. If disabled, the driver will always report a card present.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Card Detect is LOW active - Normally the driver will detect a LOW signal on this pin as card detection of the card. As there are card holders which do have HIGH signal instead, this setting can be used.
      • LDD CD - LDD (Logical Device Driver) Card Detect Interface
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Card detection pin - LDD interface to Card Detect pin
      • non-LDD CD - classic (non-LDD) Card Detect interface
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Card detection pin - Card detection pin is on low level in case a card is inserted.
    • Report 'Card present' if no Card detection pin - Sets the return value of CardPresent() in case there is no card detection pin.
    • Write protection - This enables or disables the write protection
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Write Protect is LOW active - Normally the driver will detect a LOW signal on this pin as write protection of the card. As there are card holders which do have HIGH signal for the write protection, this setting can be used.
      • LDD WP - LDD (Logical Device Driver) interface for WP (Write Protect) pin
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Write Protect Pin - Interface to Write Protect (WP) pin
      • non-LDD WP - classic (non-Logical Device Driver) Write Protect Interface
        The following items are available only if the group is enabled (the value is "Enabled"):
        • WP pin - Write protect pin, low if write protection is detected. Note that this is NOT a hardware write protection: even if the write protection is set on the card, there is no hardware way to prevent writing to the card.
  • System - System interfaces
    • Wait - Interface to Wait component
    • Timeout - Interface to timeout counters
    • RTOS - If using an RTOS, the driver will use RTOS wait in conjunction with timeouts to balance CPU cycles
      The following items are available only if the group is enabled (the value is "Enabled"):
      • RTOS - Operating system to use
